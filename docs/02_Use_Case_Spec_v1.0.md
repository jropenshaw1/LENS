# LENS Use Case Spec v1.0
**Status:** Ratified
**Date:** March 16, 2026
**Follows:** LENS Charter v1.0
**Precedes:** Data Dictionary v1.0

---

## 1. Purpose

This document defines the trigger conditions, client scope, behavioral boundaries, and reporting model for LENS v1.0. It translates the Charter's design decisions into concrete, testable scenarios that govern implementation.

---

## 2. Governing Rule — Authorship

**LENS fires when the user is the human author of the prompt.**

This is the single governing test for all scope decisions. It is authorship-based, not client-based. If the user wrote the prompt, LENS is in scope. If an AI generated the prompt (regardless of which client receives it), LENS is out of scope.

This rule future-proofs LENS against new clients and AI-to-AI workflows without requiring a maintained client list.

---

## 3. Client Scope

### 3.1 In-Scope Clients
Conversational sessions where the user is the direct prompt author:

| Client | Notes |
|--------|-------|
| Claude (web / desktop) | Primary client. All conversational sessions. |
| Gee (ChatGPT) | Conversational sessions only. |
| Lexi (Perplexity) | Conversational sessions only. |
| Grok | Conversational sessions only. |
| Gemini | Conversational sessions only. |

### 3.2 Excluded Clients
Execution environments and AI-to-AI workflows:

| Client | Reason for Exclusion |
|--------|----------------------|
| Cursor | Execution environment — prompts generated by Claude, not the user |
| Claude Code | Coding execution — not conversational prompting |
| Claude Cowork | File/task automation — not conversational prompting |
| Codex | Code generation — same pattern |
| Any AI-to-AI relay | e.g. Claude generating a prompt sent to Lexi via lexi-relay — the user is not the author |

### 3.3 Domain Exclusion — Coding Prompts
Even within in-scope clients, coding and technical execution prompts are excluded from LENS capture. The user is a leader and strategist, not a coder. Code-generation exchanges do not reflect prompt patterns in the domains that matter. Coding noise would pollute the dataset.

Agent misbehavior (e.g. Cursor schema freelancing) is governed by CLAUDE.md and OpenBrain behavioral observations — not LENS.

## 4. Trigger Conditions

LENS has two trigger types, both inherited from the Charter.

### 4.1 CLARIFICATION_TRIGGER
**Definition:** The AI asks a clarifying question because the prompt is underspecified and the AI cannot proceed without more information.

**Test:** Would the AI's response have materially changed direction without the clarification? If yes, the trigger is valid.

**Client-specific behavior:**

| Client | Trigger condition |
|--------|------------------|
| Claude | Any clarifying question required to proceed |
| Gee | Clarifying question about specificity needed to proceed — NOT Gee's trained "would you like me to show you something better?" offer |
| Lexi | Clarifying question required to proceed |
| Grok / Gemini | Any clarifying question required to proceed |

**Not a trigger:**
- Conversational follow-up questions ("tell me more")
- Confirmations of completed tasks ("does that look right?")
- Gee's trained improvement offers (not a genuine clarification need)
- Clarifying questions in coding / technical execution sessions

### 4.2 SUCCESS_PATTERN (LENS+)
**Definition:** User-invoked capture of a prompt exchange that worked exceptionally well.

**Mechanism:** The user types "LENS+" at any point in a session. The AI treats this as a signal to capture the immediately preceding exchange as a SUCCESS_PATTERN.

**Rationale:** Success patterns are too subjective for passive AI inference. User invocation ensures high-fidelity captures of exchanges the user genuinely values.

---

## 5. Capture Flow (Summary)

Full detail in Functional Spec. Summary for Use Case alignment:

1. Trigger fires (CLARIFICATION_TRIGGER or LENS+)
2. AI proposes the capture — presents: original prompt, optimized prompt, missing_dimension, delta_reason
3. User reviews and confirms, edits, or declines
4. On confirmation: AI writes to OpenBrain with full schema fields
5. On decline: no capture, no record

**User ratification is required before every capture commits to OpenBrain.**

---

## 6. Session Capture Cap

**Maximum captures per session: 5**

After 5 LENS captures in a single session, LENS goes quiet for the remainder of that session. The cap resets at the start of each new session.

**Rationale:**
- Noise has not been a problem in practice
- The user restarts sessions frequently to clear the context window, which naturally resets the cap
- Exploratory sessions with high clarification frequency should not flood the dataset

**Future consideration:** If the cap proves insufficient, a "LENS off" explicit opt-out will be evaluated for v1.1.

## 7. Happy Path — CLARIFICATION_TRIGGER

**Scenario:** User is in a Claude conversational session. User submits a prompt. Claude determines it cannot proceed without clarification. LENS fires.

**Step by step:**
1. User submits prompt: "Help me prepare for my interview next week."
2. Claude asks: "Which company and role? And is this a first-round or later-stage interview?"
3. LENS CLARIFICATION_TRIGGER fires.
4. Claude proceeds with the clarification, completes the response.
5. After response, Claude proposes capture:
   > "LENS capture proposed:
   > **Original prompt:** 'Help me prepare for my interview next week.'
   > **Missing dimension:** Company, role, and interview stage not specified.
   > **Optimized prompt:** 'Help me prepare for a [role] interview at [company]. This is a [first-round / final-round] interview. My background is [brief summary].'
   > **Delta reason:** Without company and role, preparation advice would be generic. Stage determines depth of behavioral vs. technical focus.
   > Confirm, edit, or skip?"
6. User confirms.
7. Claude writes capture to OpenBrain.

---

## 8. Happy Path — SUCCESS_PATTERN (LENS+)

**Scenario:** User has a highly productive exchange. Wants to capture it as a model prompt.

**Step by step:**
1. User completes an exchange they found valuable.
2. User types: "LENS+"
3. AI captures the immediately preceding exchange as a SUCCESS_PATTERN.
4. AI proposes the capture for ratification.
5. User confirms.
6. AI writes to OpenBrain tagged as SUCCESS_PATTERN.

## 9. Edge Cases

### 9.1 Multiple clarifications in one exchange
If an AI asks two clarifying questions before proceeding, this counts as one LENS capture — the trigger fires once per exchange, not once per question.

### 9.2 User declines ratification
No capture. No partial record. The exchange is not logged anywhere. LENS does not retry.

### 9.3 Cap reached mid-session
LENS goes quiet. No notification required unless the user asks. The user can check cap status by asking "how many LENS captures this session?"

### 9.4 AI-to-AI relay (e.g. lexi-relay)
Claude generates a prompt and sends it to Lexi. The user is not the author. LENS does not fire — in either the Claude session or the Lexi session. The authorship rule governs.

### 9.5 Ambiguous trigger — Gee's improvement offers
Gee is trained to offer prompt improvement suggestions. This is NOT a CLARIFICATION_TRIGGER. LENS only fires if Gee asks a question because she genuinely cannot proceed without more information, and her answer materially changes direction.

### 9.6 Session restart
The capture cap resets. LENS is fully active in the new session.

## 10. On-Demand Reporting

**Format:** Conversational with structured summary.

When the user requests a pattern report, the AI:
1. Presents a table ranked by frequency: ambiguity_type, count, example
2. States the impact — what these patterns cost in exchange quality
3. Opens discussion for the user to explore, question, and decide on improvements in real-time

**Rationale:** The user's preferred AI engagement model is active and conversational, not read-only. Reports are a starting point for dialogue, not a terminus.

**Heisenberg principle:** LENS is a self-modifying system. Measuring prompt behavior changes prompt behavior. The reporting model must support ongoing evolution — not lock in a static baseline. Reports should surface what changed, not just what is.
